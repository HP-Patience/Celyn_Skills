# 引导思考场景设计参考

> 从 `material-suitability-audit` skill 的"内容结构规范"中提取，作为生成场景式提问的独立参考。

## 场景三职能

| 场景 | 职能 | 设计方法 | 示例（平稳性概念） |
|---|---|---|---|
| A | 身份/本质 | "你觉得哪个更..."——迫使学习者区分本质与表象 | 平均气温 vs 气温波动程度，哪个更能概括序列特征？ |
| B | 关系/关联 | "有相关性吗？远和近有区别吗？"——暗示距离衰减 | 今天和昨天有相关性吗？今天和一个月前呢？ |
| C | 极端/后果 | "如果...会怎样？"——暴露边界条件或非平稳特性 | 涨跌完全随机，100天后不确定性比第1天大还是小？ |

## 设计原则

1. **用日常经验，不用数学**。不说"均值和方差"，说"平均气温和气温波动程度"。
2. **不给答案，给方向**。问题本身暗示正确思考方向，但结论留给学习者自己得出。
3. **每个概念配 2-3 个场景**，核心概念必配 3 个（A+B+C），辅助概念可配 2 个。
4. **场景 A 先于 B，B 先于 C**——从简单识别到复杂推理，梯度递进。
5. **场景可跨概念复用**——同一个场景（如气温记录）可以串联多个概念，形成叙事连贯性。

## 完整示例：平稳性概念

```
💭 引导思考（先不查书，用直觉回答）

场景A：你记录每天的气温。直觉上，"平均气温"和"气温波动程度"哪个
更能概括这条序列的整体特征？——这就是均值 vs 方差的意义。

场景B：今天的温度和昨天的温度，你觉得有相关性吗？今天和一个月前呢？
——联系越远越弱，这就是自协方差要度量的东西。你觉得怎么用一个数来
表达"今天和昨天有多相关"？

场景C：假如每天温度变化完全随机（涨跌不可预测），过了100天，你对
当前温度的不确定性比第1天大还是小？为什么？——这指向随机游动的方差
特性。
```

## 与教学重构版的集成

在教学重构版中，每个核心概念的结构为：

```
🔑 核心问题（一句话——这个概念回答什么问题）

💭 引导思考（场景式提问，先不查书，用直觉回答）

📖 正文（从直觉建立到形式化定义再到推导验证）
  ├── 直觉建立：从场景ABC出发，引出形式化需求
  ├── 形式定义：数学定义、公式
  ├── 推导链路：💡 因果链标注
  └── 回到场景：用刚学会的概念重新解释场景ABC

🤔 自查（合上书，用自己的话讲一遍）

🔗 前后链接（前置知识 ← 本概念 → 后续用途）
```

## 典型反面

❌ "请推导弱平稳的三个条件" —— 这是习题，不是引导思考。引导思考不该要求数学操作。

❌ "均值和方差分别度量什么" —— 太像考试题。换为场景A的"哪个更能概括"。

❌ "自协方差函数的定义是什么" —— 直接要定义。换为场景B的"怎么表达相关性远近"。
